# STM32CubeMX 完整版模板配置详解

## 一、配置前必知：GPIO模式选择原理

### 1.1 GPIO 有哪几种模式？

STM32的GPIO有8种工作模式，分为输入和输出两大类：

```
┌─────────────────────────────────────────────────────────────┐
│                      GPIO 工作模式                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  输入模式（读取外部信号）                                      │
│  ├── GPIO_MODE_INPUT           浮空输入（无上下拉）           │
│  ├── GPIO_MODE_INPUT + Pull-up   上拉输入（默认高电平）       │
│  └── GPIO_MODE_INPUT + Pull-down 下拉输入（默认低电平）       │
│                                                              │
│  输出模式（控制外部设备）                                      │
│  ├── GPIO_MODE_OUTPUT_PP       推挽输出（主动高低电平）       │
│  ├── GPIO_MODE_OUTPUT_OD       开漏输出（只能拉低）           │
│  └── GPIO_MODE_AF_PP           复用推挽（定时器、串口等）      │
│                                                              │
│  中断模式（响应外部事件）                                      │
│  ├── GPIO_MODE_IT_RISING       上升沿触发                    │
│  ├── GPIO_MODE_IT_FALLING      下降沿触发                    │
│  └── GPIO_MODE_IT_RISING_FALLING 双边沿触发                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 如何选择模式？（决策树）

```
这个引脚要做什么？
    │
    ├── 读取外部信号？
    │   ├── 信号有上拉电阻？ → 浮空输入
    │   ├── 信号是开漏输出（如Hall）？ → 上拉输入
    │   ├── 信号是推挽输出？ → 浮空输入
    │   └── 需要检测边沿事件？ → 外部中断
    │
    └── 控制外部设备？
        ├── 驱动LED/MOS管？ → 推挽输出
        ├── 接I2C总线？ → 复用开漏
        ├── 接PWM输出？ → 复用推挽
        └── 接串口TX？ → 复用推挽
```

## 二、本项目每个引脚的详细配置

### 2.1 PA0 - Hall传感器（GPIO输入，上拉）

#### 为什么这样配置？

```
Hall传感器（A3144）是开漏输出：

  ┌─────────────┐
  │   A3144     │
  │             │
  │  VCC   GND  │
  │   │     │   │
  │   └──┬──┘   │
  │      │      │
  │     OUT ────────→ PA0
  │             │
  └─────────────┘

  无磁场时：内部三极管截止，OUT悬空（高阻态）
  有磁场时：内部三极管导通，OUT拉低到GND

  如果PA0不配上拉：
    无磁场 → PA0悬空 → 电平不确定 → 可能读到0也可能读到1
    有磁场 → PA0 = 0V → 读到0

  如果PA0配上拉：
    无磁场 → PA0被上拉到3.3V → 读到1（笔不在）
    有磁场 → PA0 = 0V → 读到0（笔放入）
```

#### CubeMX配置步骤

```
1. 在芯片引脚图上找到 PA0
2. 左键点击 PA0
3. 选择 GPIO_Input
4. 左侧 System Core → GPIO → 点击 PA0
5. 右侧配置：
   - GPIO mode: Input mode
   - GPIO Pull-up/Pull-down: Pull-up  ← 关键！
   - User Label: HALL_SENSOR
```

#### 参数解释

| 参数 | 值 | 含义 |
|------|-----|------|
| GPIO mode | Input mode | 输入模式，读取外部电平 |
| GPIO Pull-up/Pull-down | Pull-up | 内部上拉，无信号时默认高电平 |
| User Label | HALL_SENSOR | 给引脚起别名，代码中可直接用 |

#### 代码中如何使用

```c
// 读取引脚电平
GPIO_PinState state = HAL_GPIO_ReadPin(HALL_SENSOR_GPIO_Port, HALL_SENSOR_Pin);

// 判断逻辑
if (state == GPIO_PIN_RESET) {  // 低电平 = 0
    // 有磁场，笔已放入
} else {  // 高电平 = 1
    // 无磁场，笔不在
}
```

### 2.2 PA1 - PWM输出（TIM2复用推挽）

#### 为什么用复用推挽？

```
PA1 被 TIM2_CH1 占用，不能当普通GPIO用。

复用（Alternate Function）= 引脚由外设控制，不由软件控制

推挽（Push-Pull）= 能主动输出高电平和低电平

PWM需要快速切换高低电平：
  __|‾‾|__|‾‾|__|‾‾|__

推挽输出速度快，驱动能力强，适合驱动MOS管H桥。
```

#### CubeMX配置步骤

```
1. 点击 PA1 → 选择 TIM2_CH1
2. 左侧 Timers → TIM2
3. Clock Source → Internal Clock
4. Channel1 → PWM Generation CH1
5. 参数配置：
   - Prescaler (PSC): 0
   - Counter Period (ARR): 654
   - Pulse (CCR1): 327
   - PWM Mode: PWM mode 1
   - CH Polarity: High
```

#### 参数计算原理

```
PWM频率 = 定时器时钟 / (PSC+1) / (ARR+1)

定时器时钟 = 72MHz（TIM2在APB1上，虽然APB1=36MHz，但定时器自动×2）

目标：110kHz
110000 = 72000000 / (0+1) / (ARR+1)
ARR+1 = 72000000 / 110000 = 654.5
ARR = 654

占空比 = CCR / (ARR+1) × 100%
初始占空比50%：CCR = 0.5 × 655 = 327
```

#### PWM mode 1 vs mode 2

```
PWM mode 1：
  计数器 < CCR 时，输出高电平
  计数器 ≥ CCR 时，输出低电平
  CCR越大，高电平时间越长，占空比越大

PWM mode 2：
  与mode 1相反
  CCR越大，占空比越小

一般用 mode 1
```

### 2.3 PA2 - 电池电压采样（ADC输入）

#### 为什么用ADC？

```
电池电压是模拟信号（连续变化的0~4.2V）
ADC把模拟电压转换成数字值（0~4095）

为什么要分压？
  电池满电4.2V，超过ADC量程（0~3.3V）
  用两个电阻分压，把电压降到安全范围

  V_bat ──[R1=10k]──┬──[R2=10k]── GND
                     │
                     └──→ PA2 (ADC输入)

  V_adc = V_bat × R2 / (R1 + R2) = V_bat × 10k / 20k = V_bat / 2

  电池4.2V → ADC测到2.1V → 在3.3V量程内 ✓
```

#### CubeMX配置步骤

```
1. 点击 PA2 → 选择 ADC1_IN2
2. 左侧 Analog → ADC1
3. 勾选 IN2（自动勾选）
4. 参数配置：
   - Scan Conversion Mode: Disabled
   - Continuous Conversion Mode: Disabled
   - Data Alignment: Right Alignment
   - Number of Conversion: 1
5. Rank 1 配置：
   - Channel: ADC_CHANNEL_2
   - Sampling Time: 239.5 Cycles
```

#### 采样时间选择原理

```
ADC转换 = 采样阶段 + 转换阶段

采样阶段：给内部电容充电
  时间太短 → 电容没充满 → 电压不准
  时间够长 → 电充满 → 电压准确

转换阶段：比较输出数字值，固定12.5个周期

总时间 = 采样时间 + 12.5个周期

239.5 Cycles → 采样时间约20μs → 精度高
1.5 Cycles → 采样时间约0.125μs → 速度快但精度低

本赛题精度要求高，选239.5 Cycles
```

### 2.4 PA3 - 电流采样（ADC输入）

#### 电流采样原理

```
电流不能直接测量，需要转换成电压：

方案：采样电阻 + 运放

  充电电流 → [0.1Ω采样电阻] → V = I × R
                                    │
                                    ▼
                              [运放 ×10倍]
                                    │
                                    ▼
                                PA3 (ADC)

  示例：
    电流1A → 电阻压降 = 1A × 0.1Ω = 0.1V
    运放放大10倍 → 1V
    ADC读取 → 1V / 3.3V × 4095 = 1241

  反推电流：
    I = ADC值 × 3.3 / 4095 / 运放增益 / 采样电阻
    I = 1241 × 3.3 / 4095 / 10 / 0.1 = 1.0A
```

#### CubeMX配置

```
与PA2类似：
1. 点击 PA3 → 选择 ADC1_IN3
2. Sampling Time: 239.5 Cycles
```

### 2.5 PA4 - 过压保护（外部中断）

#### 为什么用外部中断？

```
过压保护需要"立即响应"，不能等主循环轮询。

外部中断 = 硬件检测到边沿 → 立刻打断CPU → 执行中断函数

  正常执行代码...
       │
       ▼
  PA4检测到上升沿（电压超限）
       │
       ▼
  CPU暂停当前任务
       │
       ▼
  执行中断函数（关断充电）
       │
       ▼
  返回继续执行

响应时间：微秒级，比轮询快得多
```

#### 为什么用双边沿？

```
上升沿：低→高，检测过压发生
下降沿：高→低，检测过压解除

双边沿 = 两种都能检测
  过压发生时 → 立即关断
  过压解除时 → 可以恢复充电
```

#### CubeMX配置步骤

```
1. 点击 PA4 → 选择 GPIO_EXTI4
2. 左侧 GPIO → 点击 PA4
3. 配置：
   - GPIO mode: External Interrupt Mode with Rising/Falling edge trigger
   - GPIO Pull-up/Pull-down: No pull-up and no pull-down
   - User Label: OVP_PIN
4. 左侧 NVIC：
   - 勾选 EXTI line[4] interrupt
   - Preemption Priority: 1（数字越小优先级越高）
```

#### 中断优先级解释

```
STM32有16个优先级（0-15，数字越小越高）

  优先级0（最高）→ 系统关键中断
  优先级1 → OVP保护（本项目）
  优先级2 → 串口
  优先级15 → SysTick（最低）

高优先级中断可以打断低优先级中断：
  串口中断执行中 → OVP中断来了 → 暂停串口 → 先处理OVP → 返回继续串口
```

### 2.6 PA5 - 充电使能（GPIO输出，推挽）

#### 为什么用推挽输出？

```
充电使能引脚控制MOS管的导通/截止：

  PA5 ──→ [MOS管栅极]

  输出高电平(3.3V) → MOS管导通 → 充电模块供电 → 开始充电
  输出低电平(0V)   → MOS管截止 → 充电模块断电 → 停止充电

推挽输出：
  主动输出3.3V，驱动能力强，MOS管完全导通
  主动输出0V，MOS管完全截止

开漏输出：
  输出高电平时靠上拉电阻，驱动能力弱
  MOS管可能无法完全导通，发热严重

所以选推挽！
```

#### CubeMX配置步骤

```
1. 点击 PA5 → 选择 GPIO_Output
2. 左侧 GPIO → 点击 PA5
3. 配置：
   - GPIO output level: Low（初始关闭充电）
   - GPIO mode: Output Push Pull
   - Maximum output speed: Low（不需要高速切换）
   - User Label: CHG_EN
```

#### GPIO速度选择

```
速度选项：
  Low     → 2MHz，适合LED、使能信号
  Medium  → 10MHz
  High    → 50MHz
  Max     → 50MHz+

本引脚控制充电开关，不需要高速，选Low即可
速度越高，功耗越大，EMI干扰越大
```

### 2.8 PA9/PA10 - 串口（USART复用推挽/输入）

#### 为什么TX用推挽，RX用输入？

```
PA9 = USART1_TX（发送）
  MCU主动发送数据，需要推挽输出
  复用推挽 = 由USART外设控制引脚

PA10 = USART1_RX（接收）
  MCU接收外部数据，需要输入模式
  复用输入 = 由USART外设读取引脚
```

#### CubeMX配置步骤

```
1. 左侧 Connectivity → USART1
2. Mode → Asynchronous（异步模式）
3. 引脚自动配置：
   - PA9 → USART1_TX（复用推挽）
   - PA10 → USART1_RX（浮空输入）
4. 参数配置：
   - Baud Rate: 115200
   - Word Length: 8 Bits
   - Stop Bits: 1
   - Parity: None
   - Data Direction: Receive and Transmit
```

#### 波特率计算

```
波特率 = 每秒传输的位数

115200波特率 = 每秒传115200位

传一个字节（10位：起始+8数据+停止）需要：
  10 / 115200 ≈ 87μs

一秒能传：
  115200 / 10 = 11520字节 ≈ 11KB
```

### 2.9 PB0 - NTC温度采样（ADC输入）

#### NTC测温原理

```
NTC = 负温度系数热敏电阻
  温度升高 → 电阻降低
  温度降低 → 电阻升高

分压电路：
  VCC(3.3V) ──[NTC]──┬──[10k固定电阻]── GND
                      │
                      └──→ PB0 (ADC)

  温度高 → NTC电阻小 → 分压点电压高 → ADC值大
  温度低 → NTC电阻大 → 分压点电压低 → ADC值小

反推温度：
  R_ntc = V_adc × R_fixed / (VCC - V_adc)
  T = 1 / (1/T0 + (1/B) × ln(R_ntc/R0)) - 273.15

  其中：
    T0 = 298.15K (25°C)
    R0 = 10kΩ (25°C时的电阻)
    B = 3950 (NTC的材料常数)
```

#### CubeMX配置

```
1. 点击 PB0 → 选择 ADC1_IN8
2. Sampling Time: 239.5 Cycles
```

### 2.10 PB6/PB7 - I2C（复用开漏）

#### 为什么I2C必须用开漏？

```
I2C总线上有多个设备：

  MCU ──┬── SCL ──┬── 设备A
        │         ├── 设备B
        │         └── 设备C
        └── SDA

如果用推挽：
  设备A输出高(3.3V) + 设备B输出低(0V) = 短路！
  大电流流过，芯片烧毁！

用开漏：
  输出低 → 拉低总线
  输出高 → 松开总线，靠上拉电阻

  线与逻辑：
    所有设备都松开 → 总线高（上拉）
    任何一个拉低 → 总线低

  这样多个设备可以安全共存！
```

#### CubeMX配置步骤

```
1. 左侧 Connectivity → I2C1
2. I2C → 选择 I2C
3. 参数配置：
   - I2C Speed Mode: Standard Mode
   - I2C Clock Speed: 100000 (100kHz)
4. 引脚自动配置：
   - PB6 → I2C1_SCL（复用开漏）
   - PB7 → I2C1_SDA（复用开漏）
```

#### I2C速度选择

```
Standard Mode: 100kHz（默认，足够本项目）
Fast Mode: 400kHz（高速设备）
Fast Mode Plus: 1MHz（特殊设备）

速度越快，对上拉电阻要求越严格
100kHz时，4.7kΩ上拉即可
```

## 三、时钟配置详解

### 3.1 时钟树配置步骤

```
1. 点击顶部 Clock Configuration 标签
2. 在 Input frequency 输入 8（MHz）
3. 确认以下配置：

  HSE = 8 MHz（外部晶振）
    │
    ▼
  PLL Source = HSE
    │
    ▼
  PLL Multiplier = ×9
    │
    ▼
  SYSCLK = 72 MHz
    │
    ├──→ AHB Prescaler = /1 → HCLK = 72 MHz
    │
    ├──→ APB1 Prescaler = /2 → PCLK1 = 36 MHz
    │
    └──→ APB2 Prescaler = /1 → PCLK2 = 72 MHz
```

### 3.2 为什么APB1要分频？

```
STM32F103的总线限制：
  AHB 最大 72MHz
  APB1 最大 36MHz（硬件限制！）
  APB2 最大 72MHz

如果APB1不分频（72MHz）→ 超频 → 芯片损坏！

APB1上挂的外设：
  TIM2, TIM3, TIM4, I2C1, USART2, USART3

虽然APB1=36MHz，但定时器时钟自动×2=72MHz
所以TIM2的PWM频率不受影响
```

## 四、NVIC中断配置详解

### 4.1 配置步骤

```
1. 左侧 System Core → NVIC
2. 配置中断优先级：

  | 中断                    | 使能 | 优先级 | 说明           |
  |-------------------------|------|--------|----------------|
  | EXTI line[4] interrupt  | ✓    | 1      | OVP保护        |
  | USART1 global interrupt | ✓    | 2      | 串口           |
  | SysTick interrupt       | ✓    | 15     | 系统滴答       |
```

### 4.2 优先级分组

```
STM32有4位优先级，可以分为：
  - 0位抢占 + 4位子优先级
  - 1位抢占 + 3位子优先级
  - 2位抢占 + 2位子优先级
  - 3位抢占 + 1位子优先级
  - 4位抢占 + 0位子优先级（默认，本项目用这个）

抢占优先级：决定能否打断其他中断
子优先级：同一抢占级内的顺序

本项目用默认分组（Priority Group 4）：
  所有优先级都是抢占优先级
  数字越小，优先级越高
```

## 五、SYS配置详解

### 5.1 Debug配置

```
1. 左侧 System Core → SYS
2. Debug → Serial Wire

Serial Wire = SWD调试接口
  使用 PA13(SWDIO) 和 PA14(SWCLK)
  这两个引脚被调试器占用，不能配置为GPIO
```

### 5.2 Timebase Source

```
Timebase Source → SysTick

SysTick = 系统滴答定时器
  每1ms产生一次中断
  用于 HAL_Delay() 等延时函数
```
