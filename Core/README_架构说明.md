## 三、引脚分配表

| 引脚 | 功能 | 方向 | 说明 |
|------|------|------|------|
| PA0 | Hall传感器 | 输入(上拉) | 检测设备是否放入充电位 |

| PA3 | ADC_IN3 | 模拟输入 | 充电电流采样(0.1Ω+10倍放大) |
| PA4 | OVP中断 | EXTI上升沿 | 硬件过压保护(立即停止充电) |
| PA5 | 充电使能 | 输出(高有效) | 控制充电通路开关 |
| PA6 | 状态LED | 输出 | 指示系统运行状态 |
| PA9 | USART1_TX | 复用输出 | 调试串口发送 |

| PB6 | I2C1_SCL | 复用开漏 | BQ25601时钟线 |

## 五、核心架构: 状态机

系统的核心是一个**五状态有限状态机(FSM)**，每10ms执行一次:

```
                    Hall检测到设备
    ┌──────────────────────────────────────────┐
    │                                          ▼
┌───┴────┐  连续3次确认  ┌───────────┐   电压≥4.2V   ┌──────────┐
│  IDLE  │ ───────────→ │ DETECTING │ ──────────→ │   FULL   │
│  空闲   │              │   检测中    │              │   充满    │
└───┬────┘              └─────┬─────┘              └────┬─────┘
    │  设备移除/超时           │                         │
    │◄────────────────────────┘    设备移除/电压<4.0V   │
    │◄──────────────────────────────────────────────────┘
    │
    │  设备放入  ┌──────────┐  保护触发  ┌──────────┐
    │           │ CHARGING │ ────────→ │  ERROR   │
    │           │  充电中    │           │  错误     │
    │           └──────────┘           └────┬─────┘
    │                ▲                      │
    │                │ 设备移除              │ 设备移除
    │◄───────────────┘                      │
    └◄──────────────────────────────────────┘
```

### 各状态详细行为

| 状态 | LED | 主要行为 | 转移条件 |
|------|-----|---------|---------|
| **IDLE** | 慢闪(500ms) | 轮询Hall传感器等待设备 | Hall=1 → DETECTING |
| **DETECTING** | 快闪(100ms) | 连续3次确认设备存在 | 3次确认→CHARGING, 超时/移除→IDLE |
| **CHARGING** | 常亮 | ADC采样、功率计算、保护检查、软启动 | 移除→IDLE, 充满→FULL, 保护→ERROR |
| **FULL** | 灭 | 监控设备移除和自放电 | 移除→IDLE, V<4.0V→CHARGING |
| **ERROR** | 快闪(100ms) | 等待设备移除清除错误 | 移除→IDLE |

## 六、充电控制流程

```
CHARGING状态每10ms执行:

    ┌─────────────┐
    │ Hall检测    │ ──→ 设备移除? ──→ 停止充电, → IDLE
    └──────┬──────┘
           ▼
    ┌─────────────┐
    │ ADC采样     │ ──→ 读取电压/电流/温度(带8点滑动滤波)
    └──────┬──────┘
           ▼
    ┌─────────────┐
    │ 功率计算    │ ──→ P_out = V × I, FOD损耗 = P_in - P_out
    └──────┬──────┘
           ▼
    ┌─────────────┐
    │ 四重保护    │ ──→ OVP(4.25V) / OCP(2500mA) / OTP(80°C) / FOD(500mW)
    └──────┬──────┘     任一触发 → 停止充电 → ERROR状态
           ▼
    ┌─────────────┐
    │ 温度降额    │ ──→ T≥60°C? 降低功率到30%
    └──────┬──────┘
           ▼
    ┌─────────────┐
    │ 软启动      │ ──→ 每秒+5%功率, 20%→80%
    └──────┬──────┘
           ▼
    ┌─────────────┐
    │ 充满判断    │ ──→ V≥4.2V 且 I<50mA? → FULL状态
    └──────┬──────┘
           ▼
    ┌─────────────┐
    │ 日志输出    │ ──→ 每5秒打印V/I/T/P
    └─────────────┘
```

## 七、ADC采样与转换

### 7.1 采样方式
- **模式**: 单通道轮询(不用DMA，简单可靠)
- **滤波**: 8点滑动平均滤波，消除噪声
- **采样时间**: 239.5 ADC时钟周期(约20μs/通道)

### 7.2 物理量转换公式

| 物理量 | 公式 | 说明 |
|--------|------|------|
| 电压(V) | `Vbat = ADC×3.3/4096 × (R1+R2)/R2` | R1=R2=10k, 分压比2:1 |
| 电流(mA) | `I = ADC×3.3/4096 / (R×Gain) × 1000` | R=0.1Ω, Gain=10 |
| 温度(°C) | `T = 1/(1/T0 + (1/B)×ln(R/R0)) - 273.15` | R0=10k@25°C, B=3950 |

### 7.3 NTC温度计算原理

```
Vcc(3.3V) ───┬─── R_series(10kΩ) ───┬─── NTC(10kΩ@25°C) ───┐ GND
              │                      │                        │
              └──────────────────────┴─── ADC(PB0) ──────────┘

ADC读数 → Vadc → R_ntc = R_series × Vadc / (Vcc - Vadc)
       → T = 1/(1/298.15 + (1/3950)×ln(R_ntc/10)) - 273.15
```

## 八、安全保护机制

| 保护类型 | 阈值 | 响应方式 | 实现 |
|----------|------|---------|------|
| **OVP** 过压 | ≥4.25V | 硬件中断+软件双重检测 | EXTI4硬件中断(最高优先级) + 软件轮询 |
| **OCP** 过流 | ≥2500mA | 软件检测，立即停止 | ADC采样电流比较 |
| **OTP** 过温 | ≥80°C | 软件检测，立即停止 | NTC温度采样比较 |
| **FOD** 异物 | 功耗>500mW | 软件检测，立即停止 | 发射/接收功率差值比较 |
| **温度降额** | ≥60°C | 降低功率到30% | 软件动态调节PWM占空比 |

### 保护优先级
```
硬件OVP中断(EXTI4) > 软件OVP轮询 > OCP > OTP > FOD
```

## 十一、比赛快速调参指南

所有可调参数集中在 `main.h` 中，比赛时只需修改此文件:

```c
/* 电池参数 - 根据实际电池规格调整 */
BATTERY_FULL_VOLTAGE    // 满电电压(默认4.2V)
BATTERY_OVP_VOLTAGE     // 过压保护(默认4.25V)

/* 温度参数 - 根据散热条件调整 */
TEMP_WARN_THRESHOLD     // 降额温度(默认60°C)
TEMP_STOP_THRESHOLD     // 停止温度(默认80°C)

/* 电流参数 - 根据充电IC能力调整 */
CURRENT_OCP_MA          // 过流保护(默认2500mA)
CURRENT_CHARGE_MA       // 充电电流(默认1000mA)

/* PWM参数 - 根据发射线圈特性调整 */
PWM_ARR_VALUE           // 频率(默认654→110kHz)
CHARGE_POWER_START      // 起始功率(默认20%)
CHARGE_POWER_MAX        // 最大功率(默认80%)

/* 检测参数 - 根据机械结构调整 */
STABLE_COUNT_THRESHOLD  // 稳定次数(默认3次)
FOD_POWER_LOSS_THRESHOLD // FOD阈值(默认500mW)
```

## 十三、Qi协议说明

当前实现中Qi协议函数为**桩函数(Stub)**，仅打印调试信息:
- `Qi_SendControlError()`: 发送控制误差包
- `Qi_EndPowerTransfer()`: 结束功率传输

生产环境需要实现完整的Qi协议数据包构造和解析。充电IC BQ25601可能已内置部分Qi协议支持。
